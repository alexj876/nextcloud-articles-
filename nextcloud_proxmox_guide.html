🚀 Nextcloud on Proxmox with Cloudflare Tunnel

Guide Overview: A production-ready setup using an unprivileged LXC container on Proxmox, running Nextcloud with Nginx, PostgreSQL, Redis, and PHP-FPM — all secured behind a Cloudflare Tunnel for 24/7 external access without opening any firewall ports.
Internet → Cloudflare Tunnel → Proxmox Host → Nextcloud LXC (Nginx + PHP + PostgreSQL + Redis)
Why Cloudflare Tunnel? Zero port forwarding required. No dynamic DNS needed. Automatic HTTPS. DDoS protection included. Your home IP stays hidden.

Phase 1: Proxmox Preparation

1.1 Storage Setup (Recommended: ZFS)

Create a dedicated dataset for Nextcloud data (separate from the container root):

# On Proxmox shell
zfs create poolname/ncdata
zfs get mountpoint poolname/ncdata
# Note the mountpoint path (e.g., /poolname/ncdata)

1.2 Download Debian Template

pveam update
pveam available | grep debian
pveam download local debian-12-standard_12.7-1_amd64.tar.zst

1.3 Create the LXC Container

Via Proxmox Web UI or CLI:

pct create 110 local:vztmpl/debian-12-standard_12.7-1_amd64.tar.zst \
  --hostname nextcloud \
  --cores 4 \
  --memory 4096 \
  --swap 1024 \
  --rootfs local-lvm:64 \
  --net0 name=eth0,bridge=vmbr0,ip=192.168.1.110/24,gw=192.168.1.1 \
  --unprivileged 1 \
  --features nesting=1
Key settings:
  • Unprivileged: More secure (recommended)
  • Nesting=1: Required for some features
  • Static IP: Essential for stable internal routing

1.4 Mount External Data Storage

# Attach ZFS dataset to container
pct set 110 -mp0 /poolname/ncdata,mp=/mnt/ncdata

# Set ownership for unprivileged container (UID/GID mapping)
chown -R 100110:100110 /poolname/ncdata

1.5 Start & Enter Container

pct start 110
pct enter 110

Phase 3: Service Configuration

3.1 PostgreSQL Setup

sudo -u postgres psql <<EOF
CREATE DATABASE nextcloud TEMPLATE template0 ENCODING 'UTF-8';
CREATE USER nextcloud WITH PASSWORD 'YourStrongPasswordHere';
ALTER DATABASE nextcloud OWNER TO nextcloud;
GRANT ALL ON DATABASE nextcloud TO nextcloud;
EOF

3.2 Redis Configuration

mkdir -p /etc/redis/conf.d
cat > /etc/redis/conf.d/nextcloud.conf <<EOF
port 0
unixsocket /run/redis/redis-server.sock
unixsocketperm 770
maxclients 10240
requirepass YourRedisPasswordHere
EOF

usermod -aG redis www-data
systemctl restart redis-server

3.3 PHP-FPM Tuning

Create custom PHP configuration:

cat > /etc/php/8.4/mods-available/nextcloud.ini <<EOF
; Performance
memory_limit = 1024M
max_execution_time = 3600
default_socket_timeout = 3600
post_max_size = 16G
upload_max_filesize = 16G
output_buffering = off

; Redis Sessions
session.save_handler = redis
session.save_path = "unix:///run/redis/redis-server.sock?auth=YourRedisPasswordHere&database=1"
session.serialize_handler = igbinary
session.gc_maxlifetime = 86400
redis.session.locking_enabled = 1
redis.session.lock_retries = -1
redis.session.lock_wait_time = 10000

; OPcache
opcache.enable = 1
opcache.enable_cli = 1
opcache.jit = 1255
opcache.jit_buffer_size = 64M
opcache.memory_consumption = 256
opcache.interned_strings_buffer = 64
opcache.max_accelerated_files = 10000
opcache.revalidate_freq = 2
opcache.save_comments = 1

; APCu
apc.enable_cli = 1
apc.serializer = igbinary
apc.shm_size = 128M

; High availability
emergency_restart_threshold = 10
emergency_restart_interval = 1m
process_control_timeout = 10s
EOF

phpenmod -v 8.4 -s fpm nextcloud

Create optimized FPM pool:

cat > /etc/php/8.4/fpm/pool.d/nextcloud.conf <<EOF
[nextcloud]
user = www-data
group = www-data
listen = /run/php/php8.4-fpm-nextcloud.sock
listen.owner = www-data
listen.group = www-data
pm = dynamic
pm.max_children = 40
pm.start_servers = 8
pm.min_spare_servers = 4
pm.max_spare_servers = 16
pm.max_requests = 500
env[HOSTNAME] = $HOSTNAME
env[PATH] = /usr/local/bin:/usr/bin:/bin
env[TMP] = /tmp
env[TMPDIR] = /tmp
env[TEMP] = /tmp
EOF

systemctl restart php8.4-fpm

3.4 Nginx Configuration

rm -f /etc/nginx/sites-enabled/default

cat > /etc/nginx/sites-available/nextcloud <<'EOF'
upstream php-handler {
    server unix:/run/php/php8.4-fpm-nextcloud.sock;
}

server {
    listen 80;
    listen [::]:80;
    server_name _;

    # Security headers
    add_header Referrer-Policy                      "no-referrer"   always;
    add_header X-Content-Type-Options              "nosniff"       always;
    add_header X-Download-Options                   "noopen"        always;
    add_header X-Frame-Options                      "SAMEORIGIN"    always;
    add_header X-Permitted-Cross-Domain-Policies    "none"          always;
    add_header X-Robots-Tag                         "none"          always;
    add_header X-XSS-Protection                     "1; mode=block" always;

    # Path settings
    root /var/www/nextcloud;
    index index.php index.html /index.php$request_uri;

    client_max_body_size 16G;
    fastcgi_buffers 64 4K;
    fastcgi_connect_timeout 3600;
    fastcgi_send_timeout 3600;
    fastcgi_read_timeout 3600;

    # Enable gzip
    gzip on;
    gzip_vary on;
    gzip_comp_level 4;
    gzip_min_length 256;
    gzip_types application/atom+xml application/javascript application/json application/ld+json application/manifest+json application/rss+xml application/vnd.geo+json application/vnd.ms-fontobject application/x-font-ttf application/x-web-app-manifest+json application/xhtml+xml application/xml font/opentype image/bmp image/svg+xml image/x-icon text/cache-manifest text/css text/plain text/vcard text/vnd.rim.location.xloc text/vtt text/x-component text/x-cross-domain-policy;

    # Pagespeed (if installed)
    pagespeed off;

    location / {
        rewrite ^ /index.php;
    }

    location ~ ^\/(?:build|tests|config|lib|3rdparty|templates|data)\/ {
        deny all;
    }

    location ~ ^\/(?:\.|autotest|occ|issue|indie|db_|console) {
        deny all;
    }

    location ~ ^\/(?:index|remote|public|cron|core\/ajax\/update|status|ocs\/v[12]|updater\/.+|oc[ms]-provider\/.+|.+\/richdocumentscode\/proxy)\.php(?:$|\/) {
        fastcgi_split_path_info ^(.+?\.php)(\/.*|)$;
        set $path_info $fastcgi_path_info;
        try_files $fastcgi_script_name =404;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
        fastcgi_param PATH_INFO $path_info;
        fastcgi_param HTTPS on;
        fastcgi_param modHeadersAvailable true;
        fastcgi_param front_controller_active true;
        fastcgi_pass php-handler;
        fastcgi_intercept_errors on;
        fastcgi_request_buffering off;
    }

    location ~ ^\/(?:updater|oc[ms]-provider)(?:$|\/) {
        try_files $uri/ =404;
        index index.php;
    }

    location ~ \.(?:css|js|woff2?|svg|gif|map)$ {
        try_files $uri /index.php$request_uri;
        add_header Cache-Control "public, max-age=15778463";
        add_header Referrer-Policy "no-referrer" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header X-Frame-Options "SAMEORIGIN" always;
        add_header X-XSS-Protection "1; mode=block" always;
        access_log off;
    }

    location ~ \.(?:png|html|ttf|ico|jpg|jpeg|bcmap|mp4|webm)$ {
        try_files $uri /index.php$request_uri;
        access_log off;
    }
}
EOF

ln -s /etc/nginx/sites-available/nextcloud /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx

Phase 6: Cloudflare Tunnel Setup

This is the magic that gives you 24/7 access without port forwarding.

6.1 Install cloudflared

# Download latest cloudflared
cd /tmp
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
dpkg -i cloudflared-linux-amd64.deb

6.2 Authenticate & Create Tunnel

cloudflared tunnel login
# This opens a browser link to authenticate with your Cloudflare account
# Select the domain you want to use

cloudflared tunnel create nextcloud
# Save the tunnel ID shown

6.3 Configure Tunnel

Create config file:

mkdir -p /etc/cloudflared
cat > /etc/cloudflared/config.yml <<EOF
tunnel: YOUR_TUNNEL_ID
credentials-file: /etc/cloudflared/YOUR_TUNNEL_ID.json

ingress:
  - hostname: cloud.yourdomain.com
    service: http://localhost:80
    originRequest:
      noTLSVerify: true
  - service: http_status:404
EOF

6.4 Route DNS

cloudflared tunnel route dns YOUR_TUNNEL_ID cloud.yourdomain.com

6.5 Install as System Service

cloudflared service install
systemctl enable --now cloudflared
systemctl status cloudflared

systemctl daemon-reload

# Configure Nextcloud to use it
sudo -E -u www-data php -f /var/www/nextcloud/occ notify_push:setup https://cloud.yourdomain.com/push

Add to Nginx config (inside server block):

location /push/ {
    proxy_pass http://127.0.0.1:7867/;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_read_timeout 86400;
}

Troubleshooting

Issue Solution
Trusted domain error Add your domain to trusted_domains in config.php
Slow uploads Increase client_max_body_size in Nginx and post_max_size in PHP
Tunnel not connecting Check journalctl -u cloudflared -f for errors
Redis connection failed Verify socket permissions; ensure www-data is in redis group
Memory errors Increase LXC RAM or PHP memory_limit

Security Summary
